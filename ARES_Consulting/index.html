자문의 개요

자문 배경 및 목적

본 자문은 (주)코리아사이언스가 개발 중인 방과후 수업용 화성 탐사 로봇 교구(ARES)의 하드웨어 개발과 교육용 소프트웨어 설계 및 구현에 관한 기술 자문을 목적으로 한다.

교육적·기술적 배경

현행 교육 환경에서 텍스트 기반 코딩은 초보 학습자에게 높은 진입장벽으로 작용하며, 하드웨어와 소프트웨어를 연동하기 위해 별도 설치 및 복잡한 환경 설정이 필요하다. 국내에서도 SW 교육 의무화가 확대되고, 2025년부터 교육부의 AI 디지털교과서 도입이 추진되면서 직관적인 피지컬 컴퓨팅 교구에 대한 현장 수요가 빠르게 높아지고 있다. 아울러 글로벌 블록코딩 교육 시장에서 Scratch, MakeCode 등 기존 플랫폼은 실제 하드웨어와의 실시간 BLE 연동 기능이 제한적이어서, 브라우저 기반 비주얼 프로그래밍과 무선 하드웨어 제어를 통합하는 플랫폼의 공백이 존재한다.

ARES의 기술적 차별성

ARES 플랫폼은 다음과 같은 기술적 차별점을 통해 위의 공백을 해소한다.

  • 완전 웹 기반(설치 불필요): 별도 소프트웨어 설치 없이 Chrome·Edge 브라우저에서 즉시 실행되며, 교실 내 다양한 디바이스(PC·태블릿)를 지원한다.
  • Web Bluetooth API 실시간 연동: BLE를 통해 블록 실행 결과가 Raspberry Pi Pico 로봇에 즉각 반영되어 코딩의 인과관계를 직관적으로 경험할 수 있다.
  • 한국어 특화 인터페이스: 모든 블록·메시지·AI 어시스턴트가 한국어로 제공되어 국내 초등교육 환경에 최적화되었다.
  • 오픈 하드웨어 구조: 선택적 모듈 로드와 동적 핀 설정을 통해 학교 예산에 따라 다양한 하드웨어 구성을 하나의 소프트웨어로 지원한다.

ARES 프로젝트는 이러한 맥락에서 "화성 탐사"라는 흥미로운 주제를 통해 초등학생이 코딩과 로봇 제어를 즐겁게 익힐 수 있도록 기획되었다.

자문 범위

  • 펌웨어 설계: Raspberry Pi Pico 기반 MicroPython 펌웨어 아키텍처 설계
  • 웹 UI 설계: Web Bluetooth API와 Google Blockly를 활용한 비주얼 프로그래밍 인터페이스 설계
  • 통신 프로토콜 설계: BLE(Bluetooth Low Energy) 기반 무선 명령·응답 프로토콜 정의
  • AI 어시스턴트 설계: 초등학생 사용자 대상 한국어 LLM 기반 코딩 도우미 구조 설계
  • 코드 품질 개선: 레거시 코드 리팩토링 방향 및 모듈화 전략 자문

Raspberry Pi Pico는 저렴한 단가와 MicroPython 지원으로 교육용 로봇 플랫폼에 적합하다. RP2040 칩의 듀얼코어 구조와 충분한 GPIO 수는 서보 모터, DC 모터, 센서, LED 등 다양한 주변 장치를 동시에 제어하는 데 적합하다.

펌웨어 설계 전략

클래스 기반 모듈화

초기 절차적 스크립트 구조에서 클래스 기반 객체지향 구조로 전환하였다. 각 하드웨어 구성 요소는 독립적인 클래스(KSWheel, KSDCMotor, KSBuzzer, KSLeds, KSGun, KSDistance, KSMagSensor)로 캡슐화되어 있으며, 이를 RobotHardware 싱글톤이 통합 관리한다.

싱글톤 패턴 적용

MicroPython의 제한된 RAM(264 KB) 환경에서 하드웨어 인스턴스의 중복 생성을 방지하기 위해 RobotHardware와 SystemConfig 모두 싱글톤 패턴을 채택하였다.

전역 싱글톤 인스턴스 선언
# 전역 싱글톤 인스턴스 (모듈 로드 시 1회만 생성)
robot = RobotHardware()
sys_config = SystemConfig()

선택적 모듈 로드

하드웨어가 연결되지 않은 모듈이 있어도 펌웨어가 안정적으로 동작하도록 설계하였다. 각 모듈은 system.json의 enable_* 플래그에 따라 활성화 여부가 결정되며, 초기화 실패 시 None으로 처리하고 명령 수신 시 조용히 무시한다.

선택적 모듈 로드 패턴
if sys_config.is_module_enabled('wheel'):
    try:
        from wheel import KSWheel
        self.wheel = KSWheel()
    except Exception as e:
        self.wheel = None  # 실패해도 시스템은 계속 작동

설정의 런타임 변경

핀 번호, 모듈 활성화, 최대 속도 등의 시스템 설정을 BLE 명령(SET_PIN, SET_MODULE, SYS_SET)으로 런타임에 변경하고 system.json에 영구 저장하는 구조를 채택하였다. 교사나 관리자가 재컴파일 없이 하드웨어 구성에 맞춰 소프트웨어를 조정할 수 있다.

단일 거대 파일 대신 기능 단위로 분리된 ES6 모듈 구조를 채택하였다.

비동기 Promise 기반 통신

BLE 통신의 비동기 특성을 처리하기 위해 async/await + Promise 패턴을 채택하였다. 명령 전송 후 응답 대기를 Promise로 처리하며, 20초 타임아웃 메커니즘을 통해 응답 없는 명령에 의한 UI 블로킹을 방지한다.

블록 인터프리터 구조

Blockly 워크스페이스의 블록들을 재귀적으로 순회하며 BLE 명령으로 변환하는 인터프리터(CommandExecutor)를 구현하였다. 값 블록 평가(evaluateValueBlock)와 명령 블록 실행(processBlock)을 분리하여 수식 계산과 하드웨어 제어 로직이 명확히 구분된다.

Pico 펌웨어 모듈

main.py — 메인 애플리케이션 루프

AresRover 클래스로 구현된 메인 애플리케이션이다. 부팅 시퀀스에서 UART 버퍼 초기화 → 하드웨어 안전 정지 → 부팅 사운드 재생 → OLED "ARES READY" 메시지 출력을 순서대로 수행한다. 이후 10 ms 간격으로 메인 루프를 반복하며 UART로 수신된 명령을 처리한다.

  • UART 수신 버퍼 관리 (최대 512바이트, 오버플로우 자동 초기화)

  • 명령 처리 결과를 20바이트 단위로 분할 전송

향후 개선 방향

펌웨어 개선

버그 수정 필요 — _handle_sys_set

process_data.py의 _handle_set_module() 메서드 내부에 _handle_sys_set()의 코드가 잘못 삽입되어 있어 SYS_SET 명령이 동작하지 않는 버그가 존재한다. 해당 코드 블록을 별도 메서드로 올바르게 분리해야 한다.

미구현 — 배터리 잔량 모니터링

현재 get_battery_voltage() 메서드는 항상 100을 반환하는 미구현 상태이다. ADC로 실제 배터리 전압을 측정하고, 저전압 시 자동 정지 또는 경고음 기능을 추가해야 한다.

미구현 — 장애물 자동 회피

system.json에 collision_dist·auto_stop 설정이 정의되어 있으나, 현재 펌웨어에서 이를 활용하는 자동 정지 로직이 구현되어 있지 않다. 메인 루프에서 초음파 센서를 주기적으로 폴링하여 설정 거리 이내 장애물 감지 시 자동 정지 또는 회피 기동을 수행하는 기능이 필요하다.

개선 제안 — 비동기 모터 제어

현재 시간 제한 이동 명령(tFORWARD 등)은 time.sleep() 블로킹 호출을 사용하므로, 이동 중 다른 명령을 수신할 수 없다. MicroPython의 uasyncio 라이브러리를 활용한 비동기 태스크 방식으로 전환하면 동시 제어가 가능해진다.

웹 UI 개선

미구현 — 대시보드 기능 완성

현재 대시보드의 빠른 제어 버튼(quickCommand), 진단 실행(runDiagnostic), 캘리브레이션(startCalibration) 기능은 로그 출력만 구현된 시뮬레이션 상태이다. 실제 BLE 명령과 연동하는 기능 구현이 필요하다.

개선 제안 — 자동 재연결 및 모바일 대응

BLE 연결 중단 시 자동 재연결 로직(지수 백오프 방식)을 gattserverdisconnected 이벤트 핸들러에 추가해야 한다. 또한 Android Chrome에서의 모바일 뷰포트 최적화를 통해 교실 내 태블릿 활용성을 높여야 한다.

개선 제안 — 블록 한글화 확장

현재 일부 Blockly 내장 블록(반복문, 조건문 등)에 대한 한글 번역만 구현되어 있다. 변수·수학·논리 블록 전체에 대한 완전한 한글화와 한국어 툴팁 추가가 필요하다.

AI 어시스턴트 개선

개선 제안 — 웹 UI 통합 및 서버 배포

현재 AI 모듈은 독립 Python 스크립트로만 존재한다. EXAONE 2.4B 모델은 GPU 환경을 요구하므로, FastAPI 등을 활용하여 AI 추론 서버를 구성하고 웹 UI에서 REST API 또는 WebSocket으로 호출하는 구조가 실용적이다. 자연어 입력 → 블록 XML 자동 생성 기능도 중장기 과제로 권고한다.

중장기 기술 로드맵

PWA (Progressive Web App) 전환

현재 웹 앱을 PWA로 전환하면 홈 화면에 아이콘 추가, 오프라인 캐싱, 빠른 로딩이 가능해진다. 별도 앱 스토어 배포 없이 교사가 학생 태블릿에 즉시 배포할 수 있어 교실 운영 편의성이 크게 향상된다.

OTA (Over-The-Air) 펌웨어 업데이트

현재는 펌웨어 업데이트를 위해 USB 케이블 연결이 필요하다. BLE 또는 Wi-Fi를 통한 OTA 방식 업데이트를 도입하면 다수의 ARES 로봇을 교실에서 일괄 업그레이드할 수 있어 현장 유지보수 부담이 크게 줄어든다.

적응형 학습 분석 시스템

학생들의 블록 조합 패턴, 오류 발생 빈도, AI 어시스턴트 질의 내용을 수집·분석하면 개인 맞춤형 피드백과 수준별 블록 제안이 가능해진다. 장기적으로 학습 데이터 기반의 적응형(adaptive) 블록코딩 커리큘럼으로 발전시킬 수 있다.

VR/AR 확장 가능성

"화성 탐사" 테마와 증강·가상현실 기술의 결합은 자연스러운 확장 방향이다. ARES 로봇의 실시간 센서 데이터를 AR 오버레이로 시각화하거나, VR 환경에서 탐사 미션을 시뮬레이션하는 형태로 발전시키면 몰입형 STEAM 교육 경험을 제공할 수 있다.

표준화 및 생태계 전략

  • 웹 표준 준수: W3C Web Bluetooth API 표준 및 ECMAScript 최신 스펙 기반 개발을 유지하여 플랫폼 독립성을 확보한다.
  • 교육 콘텐츠 표준(SCORM): XML 프리셋 및 학습 콘텐츠를 SCORM(Sharable Content Object Reference Model) 호환 방식으로 구성하면 다양한 LMS(학습관리시스템)에서의 재활용이 가능하다.
  • TTA 표준 연계: 한국정보통신기술협회(TTA)의 교육용 IoT 제어 표준을 검토하여 국내 공교육 시스템과의 호환성을 강화한다.
  • 오픈소스 생태계: 핵심 Blockly 블록 정의, 펌웨어, 예제 XML 프리셋을 GitHub에 오픈소스로 공개하여 교사·개발자 커뮤니티의 자발적 콘텐츠 생산을 유도한다.

ARES 플랫폼의 블록코딩 환경은 다음과 같이 3단계 난이도의 커리큘럼으로 구조화할 수 있다.

표 13. 단계별 교육 커리큘럼
단계수준주요 활동사용 블록

미션 기반 학습 시나리오

화성 탐사 내러티브를 활용하여 미션 달성형 학습 시나리오를 구성할 수 있다.

  • 미션 1 (탐색): 정해진 경로를 자율적으로 주행하여 목적지 도달
  • 미션 2 (탐지): 자기장 센서로 '화성 광물(자석)'을 탐지하고 LED로 신호 발신
  • 미션 3 (회피): 초음파 센서로 장애물을 감지하고 우회 경로로 복귀
  • 미션 4 (전투): 자기장 센서로 목표물 탐지 후 BB탄 발사
  • 미션 5 (통합): 위 기능을 모두 조합한 자율 탐사 프로그램 작성

미션 결과를 OLED에 표시하고 부저로 성공 알림을 출력하는 방식으로 즉각적인 성취감을 제공한다.

하드웨어 구성별 제품 라인업

선택적 모듈 로드 구조(enable_*)와 동적 핀 설정(SET_PIN) 기능을 활용하여 교구 구성을 유연하게 변경할 수 있다.

  • 기본형: 이동 + 센서 (가장 저렴한 구성)
  • 표준형: 이동 + 센서 + LED + 부저 + OLED
  • 완전형: 모든 모듈 포함 (BB탄 발사기 포함)

학교 예산과 교육 목표에 따라 동일 소프트웨어로 다양한 하드웨어 구성을 지원할 수 있다.

교사 지원 도구 강화

현재 대시보드 기능을 확장하여 교사 전용 관리 패널을 구현하면 수업 운영 효율이 크게 향상된다.

  • 다중 로봇 관리: 여러 ARES 로봇의 연결 상태를 한 화면에서 모니터링
  • 학습 진도 추적: 학생이 저장한 XML 파일을 수집·분석하여 블록 사용 패턴 파악

  • STEAM 융합교육: "화성 탐사"라는 스토리텔링 주제는 과학(행성·궤도)·기술(로봇 제어)·공학(회로·센서)·수학(좌표·비율)을 자연스럽게 융합하는 STEAM 교육 모델에 직접 부합한다.
  • 방과후학교 및 메이커교육: 코딩학원·메이커스페이스·방과후학교 등 비정규 교육 시장에서 실제 하드웨어를 제어하는 체험형 교구는 텍스트 기반 강의 대비 높은 학습 몰입도를 제공한다.

main.py — 메인 애플리케이션 루프
# ARES 로버 메인 애플리케이션

from machine import UART, Pin
import utime
from process_data import CommandProcessor
from hardware import robot
from pins import UART_TX_PIN, UART_RX_PIN

# UART 설정
UART_ID = 0
UART_BAUDRATE = 9600

# 루프 설정
MAIN_LOOP_DELAY_MS = 10
RECEIVE_TIMEOUT_MS = 50
MAX_BUFFER_SIZE = 512


class AresRover:
    """ARES 로버 메인 애플리케이션 클래스"""

    def __init__(self):
        # UART 초기화 (블루투스 통신)
        self.uart = UART(
            UART_ID,
            baudrate=UART_BAUDRATE,
            tx=Pin(UART_TX_PIN),
            rx=Pin(UART_RX_PIN)
        )
        # 명령 프로세서 초기화
        self.processor = CommandProcessor()
        # 하드웨어 싱글톤 참조
        self.robot = robot
        # 실행 상태
        self.is_running = False
        # 수신 버퍼
        self.rx_buffer = ""

    def boot(self):
        """부팅 시퀀스 실행"""
        if self.uart.any():
            print("Cleaning UART buffer...")
            while self.uart.any():
                self.uart.read()
        self.rx_buffer = ""

        self._safe_stop_all_motors()
        self.robot.buzzer.boot_sound()

        self.robot.oled.fill(0)
        self.robot.oled.text("ARES READY", 0, 0)
        self.robot.oled.show()

        print("ARES Rover: Waiting for data...")
        self.is_running = True

    def _safe_stop_all_motors(self):
        """모든 하드웨어 안전 정지"""
        self.robot.stop_all()

    def _read_uart_line(self):
        """UART에서 완전한 라인 읽기"""
        if not self.uart.any():
            return None

        while self.uart.any():
            chunk = self.uart.read()
            if chunk:
                self.rx_buffer += chunk.decode('utf-8', 'ignore')

            if len(self.rx_buffer) > MAX_BUFFER_SIZE:
                print("[UART] 버퍼 오버플로우, 초기화")
                self.rx_buffer = ""
                return None

            utime.sleep_ms(5)

        utime.sleep_ms(RECEIVE_TIMEOUT_MS)

        while self.uart.any():
            chunk = self.uart.read()
            if chunk:
                self.rx_buffer += chunk.decode('utf-8', 'ignore')

        if '\n' in self.rx_buffer:
            newline_idx = self.rx_buffer.index('\n')
            complete_line = self.rx_buffer[:newline_idx].strip()
            self.rx_buffer = self.rx_buffer[newline_idx + 1:]
            return complete_line

        if self.rx_buffer:
            data = self.rx_buffer.strip()
            if data and not data.endswith(','):
                self.rx_buffer = ""
                return data

        return None

    def _is_status_message(self, data):
        """블루투스 상태 메시지 확인"""
        return data.startswith('+')

    def _send_response(self, response):
        """UART로 응답 전송 (20바이트 청킹)"""
        data = response.encode('utf-8') + b'\n'
        CHUNK_SIZE = 20

        for i in range(0, len(data), CHUNK_SIZE):
            chunk = data[i:i + CHUNK_SIZE]
            self.uart.write(chunk)
            utime.sleep_ms(10)

    def _process_command(self, data):
        """명령 처리 및 응답 반환"""
        try:
            print(f"[CMD] {data}")
            result = self.processor.process(data)
            return f"{result}"
        except Exception as e:
            print(f"처리 오류: {e}")
            return "ERROR"

    def run(self):
        """메인 루프 실행"""
        self.boot()

        while self.is_running:
            try:
                data = self._read_uart_line()

                if not data:
                    pass
                elif self._is_status_message(data):
                    print(f"BT 상태: {data}")
                else:
                    response = self._process_command(data)
                    self._send_response(response)

            except Exception as e:
                print(f"UART 오류: {e}")

            utime.sleep_ms(MAIN_LOOP_DELAY_MS)


if __name__ == "__main__":
    app = AresRover()
    app.run()

# ARES 로버 명령 처리기

import utime
import time
from hardware import robot
from system_config import sys_config

PWM_MAX = 65535


class CommandProcessor:
    """UART 명령 파싱 및 실행 클래스"""

    def __init__(self):
        pass

    def _get_speed_pwm(self):
        """설정된 max_speed에서 PWM 값 계산"""
        spd = sys_config.get('max_speed') / 100.0
        return int(spd * PWM_MAX)

    def process(self, data):
        """명령 처리 및 결과 반환"""

        if data == "PING":
            return self._handle_ping()
        elif data == "READY":
            return "READY"
        elif data == "STOP_ALL":
            return self._handle_stop_all()
        elif data == "GET_STATUS":
            return self._handle_get_status()
        elif data == "GET_SYS":
            return self._handle_get_sys()
        elif data == "GET_MODULES":
            return self._handle_get_modules()
        elif data.startswith("SYS_SET"):
            return self._handle_sys_set(data)
        elif data.startswith("SET_PIN"):
            return self._handle_set_pin(data)
        elif data.startswith("SET_MODULE"):
            return self._handle_set_module(data)
        elif data.startswith("BUZZER_ON"):
            return self._handle_buzzer_on(data)
        elif data.startswith("SING"):
            return self._handle_sing()
        elif data == "DISTANCE":
            if robot.distance_sensor:
                dist = robot.distance_sensor.get_distance()
                return f"DIST:{dist}"
            return "DIST:ERROR"
        elif data == "MAGNET":
            if robot.mag_sensor:
                mag = robot.mag_sensor.detect()
                return f"MAG:{mag}"
            return "MAG:ERROR"
        # ... (서보/DC 모터/LED/디스플레이 명령 핸들러 생략)

        return 0
hardware.py — 하드웨어 싱글톤 초기화
# ARES 로버 하드웨어 싱글톤

from machine import I2C, Pin, UART
from system_config import sys_config
import time
import gc
import machine


class RobotHardware:
    """하드웨어 싱글톤 클래스"""
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super(RobotHardware, cls).__new__(cls)
            cls._instance._init_hardware()
        return cls._instance

    def _init_hardware(self):
        """하드웨어 초기화 - 모듈별 선택적 로드"""
        self.temp_sensor = machine.ADC(4)
        self.start_time = time.ticks_ms()

        # 모든 모듈을 None으로 초기화
        self.dcmotor = self.oled = self.i2c = self.buzzer = None
        self.distance_sensor = self.wheel = self.mag_sensor = None
        self.leds = self.gun = None

        # 각 모듈을 enable_* 플래그에 따라 선택적으로 로드
        if sys_config.is_module_enabled('dcmotor'):
            try:
                from dcmotor import KSDCMotor
                self.dcmotor = KSDCMotor()
                self.dcmotor.dc_stop()
            except Exception as e:
                print(f"[HW] DC 모터 초기화 실패: {e}")

        if sys_config.is_module_enabled('oled'):
            try:
                from ssd1306 import SSD1306_I2C
                i2c_sda = sys_config.get('pin_i2c_sda')
                i2c_scl = sys_config.get('pin_i2c_scl')
                self.oled = SSD1306_I2C(128, 64, scl_pin=i2c_scl, sda_pin=i2c_sda)
            except Exception as e:
                print(f"[HW] OLED 초기화 실패: {e}")

        if sys_config.is_module_enabled('wheel'):
            try:
                from wheel import KSWheel
                self.wheel = KSWheel()
                self.wheel.stop()
            except Exception as e:
                print(f"[HW] 서보 모터 초기화 실패: {e}")

        # ... (나머지 모듈 초기화 생략)

        self._sync_bt_name()

    def get_status_dict(self):
        """시스템 상태 딕셔너리 반환"""
        status = {
            "dist": 0, "mag": 0,
            "temp": self.get_temperature(),
            "mem": gc.mem_free(),
            "uptime": int(time.ticks_diff(time.ticks_ms(), self.start_time) / 1000)
        }
        if self.distance_sensor:
            try: status["dist"] = self.distance_sensor.get_distance()
            except: pass
        if self.mag_sensor:
            try: status["mag"] = self.mag_sensor.detect()
            except: pass
        return status

    def stop_all(self):
        """모든 하드웨어 안전 정지"""
        for hw, stop_fn in [
            (self.dcmotor, lambda: self.dcmotor.dc_stop()),
            (self.wheel,   lambda: self.wheel.stop()),
            (self.leds,    lambda: self.leds.leds_off()),
        ]:
            if hw:
                try: stop_fn()
                except Exception as e: print(f"[stop_all] 오류: {e}")


# 전역 싱글톤 인스턴스
robot = RobotHardware()

    "pin_leds": [20, 19, 18, 17, 16],
}


class SystemConfig:
    """시스템 설정 싱글톤"""
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            cls._instance = super(SystemConfig, cls).__new__(cls)
            cls._instance._load()
        return cls._instance

    def _load(self):
        try:
            with open(CONFIG_FILE, 'r') as f:
                self.config = json.load(f)
            # 누락된 기본값 채우기
            for key, value in DEFAULT_CONFIG.items():
                if key not in self.config:
                    self.config[key] = value
        except:
            self.config = dict(DEFAULT_CONFIG)
            self._migrate_legacy()
            self.save_all()

    def get(self, key, default=None):
        return self.config.get(key, default)

    def is_module_enabled(self, module_name):
        return bool(self.config.get(f'enable_{module_name}', 0))

    def save_all(self):
        """수동 JSON 포맷팅으로 저장 (MicroPython 호환)"""
        try:
            with open(CONFIG_FILE, 'w') as f:
                f.write('{\n')
                items = list(self.config.items())
                for i, (k, v) in enumerate(items):
                    comma = ',' if i < len(items) - 1 else ''
                    if isinstance(v, str):
                        f.write(f'  "{k}": "{v}"{comma}\n')
                    elif isinstance(v, list):
                        f.write(f'  "{k}": {json.dumps(v)}{comma}\n')
                    else:
                        f.write(f'  "{k}": {v}{comma}\n')
                f.write('}\n')
        except Exception as e:
            print(f"설정 저장 오류: {e}")


# 전역 싱글톤
sys_config = SystemConfig()
